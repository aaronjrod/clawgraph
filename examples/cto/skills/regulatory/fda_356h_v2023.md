# FDA Form 356h: NDA Admin Package Standards (v2023)

## Section 1: Overview
This document specifies the required structure and content for a high-fidelity NDA Administrative Package following the latest FDA 356h Form revisions (July 2023).

## Section 2: Core Components

### 2.1 Establishment Information (Field 28)
- **Establishment Name & Address**: Full legal name and physical location.
- **FEI Number**: FDA Establishment Identifier.
- **DUNS Number**: Data Universal Numbering System.
- **Manufacturing Steps**: List specific activities (e.g., API synthesis, finished drug product manufacturing, primary packaging, sterilisation).
- **Type of Testing**: QC release, stability, microbiological.

### 2.2 Cross References (Field 30)
- List all related **INDs (Investigational New Drugs)**, **DMFs (Drug Master Files)**, and **MAFs (Manufacturing Facility Master Files)**.
- Format: `[Application Type] [Number] - [Holder Name]`.

### 2.3 Regulatory Cover Letter
- Address to the relevant FDA Division (e.g., CDER/OND).
- Explicit statement of application type (NDA 505(b)(1)).
- Proposed Indication(s).

## Section 3: Formatting Standards
- Use **BOLD HEADERS** for Field IDs.
- Ensure all FEI numbers follow the 10-digit standard.
- Tables should be used for establishment matrices.
